How it works

From pasted URL to conversion-ready AI in six steps

SiteCloser is designed to feel effortless on the surface and deeply engineered underneath. Here is the whole flow, from crawl to live widget.

1

Paste your website URL

The dashboard validates the root URL, creates a site record, and starts a background processing task so setup never blocks the UI.

2

Crawl the live website

Playwright loads the real rendered site, waits for JavaScript-driven content, follows internal links, and skips admin, checkout, login, and noisy URLs.

3

Extract content and UI structure

Trafilatura and BeautifulSoup extract readable content while a dedicated UI extractor finds nav links, buttons, search bars, logos, dropdowns, and footers.

4

Chunk and embed

Pages become semantically meaningful chunks with heading breadcrumbs. Those chunks are embedded, stored, and loaded into memory when chat starts.

5

Answer with retrieval + sales prompting

When a buyer asks something, SiteCloser retrieves the strongest chunks, classifies intent, and answers in a grounded sales-assistant tone instead of generic summary mode.

6

Deploy the widget

Copy one script tag into any page. The widget streams responses, shows starter prompts, suggests follow-ups, and links the visitor to the most relevant source page.

Under the hood

Technical enough for developers, simple enough for founders

Crawl + extraction

  • Async Playwright browser crawl.
  • Trafilatura-first content extraction.
  • HTML-aware fallback parsing when needed.

Retrieval pipeline

  • Semantic chunking with section context.
  • Embeddings + in-memory vector loading.
  • MMR reranking for diverse top results.

Conversation layer

  • SSE token streaming for responsive chat.
  • Session memory for follow-up questions.
  • OpenRouter-powered answer generation.
