How it works

A customer-facing launch flow on the surface, with solid retrieval under the hood

5minbot is designed so the setup feels approachable for a first-time user while still doing the heavier work needed for grounded answers and clean widget behavior.

The customer journey

What the launch experience feels like

1

Add your website

Paste a public URL and start the first run from a guided workspace instead of a bare admin screen.

2

Watch training progress

See clear status changes, counts, and next steps while 5minbot crawls, extracts, and prepares the chatbot.

3

Test before launch

Use the built-in chat lab to check real questions before you ever paste the widget on your site.

1

Normalize the site and start background processing

The workspace validates the URL, creates the site record, and starts a background task so the interface stays responsive while processing begins.

2

Crawl the rendered site

Playwright loads the real website, follows internal links, and waits for JavaScript content so 5minbot sees the same surface your visitors see.

3

Extract text and UI structure

Readable content is extracted from each page while a dedicated UI layer identifies navigation, search, buttons, logos, and other layout cues.

4

Chunk, embed, and store site knowledge

Pages are turned into semantic chunks with section breadcrumbs, then embedded so retrieval can surface the right answer quickly when chat starts.

5

Answer with retrieval and response strategy

When a visitor asks something, 5minbot retrieves relevant chunks, recognizes the intent, and answers in a concise product-aware voice.

6

Launch the widget

Once the site is ready, you copy one embed tag, place it on your site, and let the widget handle the first layer of buyer questions live.

Under the hood

The implementation detail that actually builds confidence

Crawling and extraction

  • Async browser crawl with Playwright.
  • Primary extraction through Trafilatura.
  • Fallback parsing for noisy pages.

Retrieval quality

  • Semantic chunking with breadcrumb context.
  • Embeddings and reranking for stronger recall.
  • Dedicated UI layout chunks for where-to-click questions.

Conversation layer

  • Streaming responses over SSE.
  • Starter prompts and follow-up suggestions.
  • Session memory for cleaner multi-turn context.

Launch workflow

  • Production-ready embed tag generation.
  • Built-in test chat before launch.
  • Clear retry and reprocess flows when needed.
