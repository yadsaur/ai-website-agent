How 5minBot handles website and chat data

This page explains the current hosted product behavior in plain language. It is intended to help teams understand what the product stores and why.

Website content you choose to process

When you add a site, 5minBot crawls public pages from the URL you provide so it can build a chatbot around that content.

Stored product data

The hosted app stores site records, processed pages, content chunks, and vector data required to answer questions inside the product.

Chat interactions

Short-lived session memory is used to make follow-up answers more coherent inside the widget and workspace test chat.

Third-party services

The current hosted product uses managed services to run the application, store data, generate embeddings, and produce model responses.

Public-site assumption

5minBot is designed for public website content. Teams should avoid processing pages that contain confidential or regulated information they should not send to hosted services.

Questions about data handling

If you need clarification on a specific deployment or data path, use the support page to contact the team directly.
