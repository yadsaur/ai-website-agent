How 5minBot handles website and chat data

This page explains the current hosted product in plain language so teams can understand what is processed, what is stored, and what kind of content the product is designed for.

Public site content

When you add a site, 5minBot crawls public pages from the URL you provide so it can build a chatbot around that content.

Stored product data

The hosted app stores site records, processed pages, content chunks, and supporting vector data required to answer questions inside the product.

Chat interactions

Short-lived conversational context is used to make follow-up answers more coherent inside the widget and workspace chat lab.

Product boundaries

5minBot is intended for public website content. Teams should avoid using the hosted product for sensitive material they should not send to hosted services.
