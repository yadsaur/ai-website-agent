How we built an AI that understands website UI layout

Plain text extraction is not enough when visitors ask where the search bar is, which tab holds pricing, or where the footer links live.

Most retrieval systems flatten a page into plain text. That works for copy-heavy questions, but it breaks the moment a visitor asks something about location or interface. “Where is the search bar?” is not a text question. It is a structure question.

What we extract beyond plain text

  • Navigation links and their inferred location.
  • Buttons, CTAs, logos, search inputs, and dropdown menus.
  • Footer links and structural landmarks like header, sidebar, or hero section.

Those UI facts get transformed into natural-language guide text and embedded as dedicated chunks. During retrieval, UI-position queries get a score bonus so those chunks surface first. The language model then receives explicit instructions to answer layout questions from that UI context rather than claiming ignorance.

The result is a chatbot that can guide a visitor through the website instead of only summarizing what the website says.
